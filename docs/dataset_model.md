To facilitate comparability across different combinations of datasets, modules, and parameter configurations within the context of CATS, a certain degree of dataset normalization is required.
Different approaches have been discussed at length as part of a master thesis written by one of the project authors (available upon request), with the results being presented here.

## Structure and Contents

The data structure itself, per alert, is realized in JSON, consisting of three top-level fields:
- `full_alert`: The full, unaltered alert as it was originally present.
- `features`: A set of standardized features, extracted from the full alert, intended to enable different alert triage methods.
- `metadata`: Information not inherently contained in each alert, but still tied to it.

As the contents and structure of `full_alert` can and will differ wildly between alert sources, `features` and `metadata` are designed to facilitate processing these alerts in a standardized, automated way.
Their contents are described in the following.

### Features
"Mandatory" means that this information should be provided by every alert, no matter it's source.
Naturally, most features are optional, as many values are only present in certain types of alerts.
| Field Name            | Data Type      | Mandatory? | Description                                                                                                                                                                                                                                                                                                                                                      |
| --------------------- | -------------- | ---------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| timestamp             | string         | **Yes**    | Contains the timestamp of the alert in the ISO 8601 format, in UTC time.<br>Should the original timestamp be in another format (like Epoch), it is converted accordingly.<br>Some alerts may feature multiple timestamps (e.g., one for the original event, one for the time the alert was created).<br>In this case, only the earliest timestamp is considered. |
| event_code            | integer        | No         | A code identifying the underlying event of the alert.<br>Notably, this is not a unique identifier for a specific event, but rather a generic code assigned to a given type of event (e.g., file creation).                                                                                                                                                       |
| event_source          | string         | No         | The file or provider a given event was collected from.<br>For Windows events, this should be a provider such as "Microsoft-Windows-Sysmon", while for Linux events this will usually be a path like "/var/log/mail.info"                                                                                                                                         |
| hostname              | string         | No         | The name of the host this event originated from.                                                                                                                                                                                                                                                                                                                 |
| username              | string         | No         | The name of the user associated with this event.                                                                                                                                                                                                                                                                                                                 |
| log_level             | string         | No         | The level assigned to this event, such as "Information" or "Error".                                                                                                                                                                                                                                                                                              |
| source_ip             | string         | No         | Source IP address associated with this event, in dotted quad notation.                                                                                                                                                                                                                                                                                           |
| destination_ip        | string         | No         | Destination IP address associated with this event, in dotted quad notation.                                                                                                                                                                                                                                                                                      |
| source_port           | integer        | No         | Source port associated with this event.                                                                                                                                                                                                                                                                                                                          |
| destination_port      | integer        | No         | Destination port associated with this event.                                                                                                                                                                                                                                                                                                                     |
| protocol              | string         | No         | The protocol associated with this (usually network) event, such as TCP or ssh.                                                                                                                                                                                                                                                                                   |
| rule_name             | string         | No         | The name of the rule which generated the present alert.                                                                                                                                                                                                                                                                                                          |
| rule_id               | string         | **Yes**    | The ID of the rule which generated the present alert.                                                                                                                                                                                                                                                                                                            |
| rule_level            | string         | No         | The level of the rule which generated the present alert.<br>Their meaning is heavily dependent on the alert source in question and will usually require some further normalization.                                                                                                                                                                              |
| rule_category         | list[*string*] | No         | The category or categories assigned to the rule which generated the present alert.                                                                                                                                                                                                                                                                               |
| rule_attacktechniques | list[*string*] | No         | Contains one or more MITRE ATT&CK technique IDs associated with a given alert, or rather the rule that has led to its generation.<br>These IDs are always of the form `tXXXX`, regardless of whether it originally described a sub-technique or not (e.g., `t1234.567` will be shortened to `t1234`).                                                            |
| rule_attacktactics    | list[*string*] | No         | Contains one or more MITRE ATT&CK tactics associated with a given alert, or rather the rule that has led to its generation.                                                                                                                                                                                                                                      |


### Metadata
All metadata fields are mandatory.
| Field Name   | Data Type      | Description                                                                                                                                                                                                                                                                                                                                                                                                           |
| ------------ | -------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| misuse       | bool OR string | Indicates whether this alert is a false positive (value will be `false`) or a true positive (value will be `true`).<br>Alternatively, this can also be one of the strings `yes`/`no`/`unknown`, in case a 3-class label is needed.                                                                                                                                                                                    |
| alert_source | string         | Lists the source of this alert in the form of dataset and alert framework, for example `ait_wazuh` or `socbed_sigma`.                                                                                                                                                                                                                                                                                                 |
| alert_index  | integer        | An ongoing index assigned to each alert, starting at 1. Alerts are counted individually for each source.                                                                                                                                                                                                                                                                                                              |
| alert_id     | string         | A unique identifier generated with `uuid4` for each alert during the normalization process.<br>This identifier is seeded (using `alert_source + str(alert_index)` as the seed), so it will not change upon recreating the dataset.<br>It is mostly intended to enable proper indexing of any combination of normalized alert datasets for frameworks like `pandas`, granting substantial speed-ups during processing. |
